# Exec plan: visual environment evidence

## Goal

Add honest visual smoke evidence for environment lighting:

- HDRI reflection uses a smooth metallic material, not a matte cube.
- Skylight/ambient readability uses a table, an object above it, and an object
  under it, with key-light shadows available so the lower object can be darker
  while still visible through environment lighting.

## Assumptions

- `flutter_scene` exposes `DirectionalLight.castsShadow` and related shadow-map
  controls; exposing a narrow viewer-controlled shadow flag is still adapter
  work, not a custom renderer.
- Runtime arbitrary HDRI download is not the preferred product workflow. Apps
  should normally present a small curated set such as forest, coast, city, and
  studio as app-bundled assets or explicit descriptors.
- The current Poly Haven source remains useful for explicit opt-in smoke/demo
  checks, but should not be described as the normal default runtime path.

## Non-goals

- Do not add a benchmark harness.
- Do not bundle large HDRI assets in this slice.
- Do not implement imported glTF lights or authored scene playback.
- Do not add a custom sky shader, custom PBR renderer, or fake baked shadows.

## Steps

1. Change: expose narrow studio key-light shadow controls through
   `ViewerLighting` and the adapter-neutral lighting frame.
   Verify: widget/render-surface tests observe the flag and runtime adapter
   maps it to `DirectionalLight.castsShadow` plus upstream shadow-map quality
   fields.
2. Change: add a compact generated GLB fixture with a table, one upper object,
   and one lower object.
   Verify: fixture generator runs and model loader/part registry tests can
   inspect the fixture.
3. Change: document the distinction between HDRI reflection evidence,
   skylight/shadow-readability evidence, and curated HDRI preset workflows.
   Verify: focused docs and fixture tests plus full repo harness.

## Acceptance criteria

- [x] HDRI reflection smoke guidance uses a metallic low-roughness material;
- [x] skylight smoke fixture contains table, upper object, and lower object;
- [x] studio lighting can enable and tune upstream key-light shadows without
      exposing concrete `flutter_scene` types;
- [x] docs do not present arbitrary runtime HDRI download as the default
      product workflow;
- [x] verification log records focused checks and full harness result.
