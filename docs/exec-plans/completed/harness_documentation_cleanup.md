# Exec plan: Harness documentation cleanup

## Goal

Leave one concise agent entry point, remove documentation that duplicates a
canonical source or preserves already-promoted planning history, and make the
repository-native checks match the actual ExecPlan lifecycle.

## Assumptions

- The user's untracked
  `docs/exec-plans/deferred/028_adaptive_ray_path_tracing_feasibility.md` is
  unrelated work and must remain untouched.
- No project-owned `.tr.md`, `.md.tr`, or `.tr` documentation exists in the
  current tree; ignored dependency locale data is outside this cleanup.
- This repository uses its existing lightweight ExecPlan format rather than
  adopting the harness skill's incompatible managed plan schema.
- The requested outcome is a proportional repair and assessment, not
  `harness-ready` certification or production attestation.

## Non-goals

- Do not change product behavior or public APIs.
- Do not add the full harness scaffold, certification records, or CI.
- Do not rewrite completed plan history or vendored third-party documentation.

## Steps

1. Change: audit instructions, documentation routes, and generated clutter.
   Verify: record exact duplicate/orphan targets and preserve unrelated work.
2. Change: trim `AGENTS.md`, retire redundant documents, and align repository
   lint with an empty-or-single-active-plan lifecycle.
   Verify: repository searches show no live references to retired documents.
3. Change: prevent Python cache recurrence and update harness/tooling contracts.
   Verify: run repository-native checks and the external adaptive harness check.

## Acceptance criteria

- [x] `AGENTS.md` is a concise map with accurate plan and verification routes.
- [x] Redundant or already-promoted documentation is removed without losing a
      unique product, architecture, or operational constraint.
- [x] A clean checkout may have zero active plans, while more than one active
      plan fails repository lint.
- [x] Project-owned `.tr` documentation is absent.
- [x] Generated Python cache files are ignored and the observed cache is gone.
- [x] `bash tools/run_checks.sh`, `python3 tools/repo_lint.py`, and
      `git diff --check` are run; scoped pre-existing failures are recorded
      literally rather than blended with this change's evidence.
