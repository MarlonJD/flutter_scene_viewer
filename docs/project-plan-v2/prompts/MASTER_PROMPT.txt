PRODUCT PURPOSE

Build a high-level, community-maintained, Flutter-native static GLB viewer and product configurator on top of `flutter_scene`.

The package must load GLB models from network/assets/bytes, preserve transform-only assembly/sub-assembly nodes, index mesh primitives as stable parts, provide camera and picking controls, change PBR materials/textures at runtime, reset originals, persist descriptor state, cache resources, and stop rendering while idle.

This is NOT a new 3D engine.

NON-NEGOTIABLE ENGINE BOUNDARY

1. Build on `flutter_scene`; do not build a raw `flutter_gpu` renderer.
2. Do not write a new PBR master shader or GLSL implementation for the MVP.
3. Use the existing `flutter_scene` GLB runtime importer, scene graph, PBR material, lighting, raycast, bounds, texture helpers, and rendering APIs.
4. Keep all upstream calls behind a narrow adapter.
5. Do not expose raw upstream Node, Material, or GPU texture objects as the primary public API.
6. If an essential public capability is missing, first make a minimal spike and propose a small upstream PR. Do not silently start a fork or engine rewrite.

V1 SCOPE

- Static single-file GLB.
- ModelSource: asset, network, bytes.
- Triangle primitives already contained in the GLB.
- Transform-only nodes as assembly/sub-assembly/dummy objects.
- Stable PartAddress: deterministic child-index node path plus primitive index.
- Duplicate node names and multiple primitives.
- Viewer-controlled studio environment/IBL, exposure, background, and optional directional light.
- Perspective camera, orbit, pan, zoom, bounds fit.
- Raycast/picking and visibility.
- Standard glTF metallic-roughness PBR.
- Runtime base-color texture and base color/metallic/roughness/emissive factors.
- Preserve original normal, metallic-roughness, occlusion, alpha, and double-sided behavior.
- Runtime normal/MR/occlusion texture slots only if the real upstream API supports them reliably; expose capability flags.
- Original material snapshot/reset and shared-material copy-on-write.
- Serializable state descriptors, model and texture caching.
- Adaptive/on-demand rendering.
- Typed diagnostics and input limits.

EXPLICIT V1 NON-GOALS

- No tessellation. GLB is already tessellated into triangle/index/accessor data.
- No UV unwrap, tangent generation, mesh repair, simplification, CAD conversion, or DCC-specific axis guessing.
- No STEP, IGES, DWG, FBX, OBJ.
- No skeletal animation, interactive bone posing, skinning controller, morph targets, blend shapes, cloth, or deformation.
- Rigid/node animation is post-MVP and must not enter initial acceptance criteria.
- No embedded glTF camera import or KHR_lights_punctual in V1; use viewer-controlled lighting.
- No Draco, meshopt, KTX2/BasisU runtime decode in V1.
- No parallax, displacement, SSS, world-aligned texture, VR, AR, physics, game engine, or runtime arbitrary shader compilation.

GLB AND GEOMETRY RULES

- Pass GLB bytes to the upstream runtime importer.
- Do not compute topology or tessellate.
- Build viewer diagnostics from imported capabilities.
- Texture assignment requires an appropriate UV set. If missing, return a typed `MissingUvSet` result and preserve the previous material.
- Do not auto-unwrap UVs.
- Do not generate tangents in the viewer. Document the actual upstream tangent/normal-map behavior and add visual fixtures.
- Do not guess source-DCC coordinate systems. Use upstream canonical glTF-to-engine conversion and provide only an explicit optional root transform escape hatch.
- Unsupported topology/extensions must be surfaced through typed diagnostics, not silently advertised as supported.

SCENE GRAPH AND PART RULES

- Preserve nodes without meshes as assembly/sub-assembly/dummy transform nodes.
- Traverse the imported tree once and build a deterministic registry.
- Address each mesh primitive independently.
- Node names are labels, not identity.
- Canonical identity is child-index path + primitive index, scoped to a model fingerprint.
- Capture material usage to detect shared instances.
- Prove with a fixture that changing one shared-material primitive does not affect another.

MATERIAL AND TEXTURE RULES

- Use existing `PhysicallyBasedMaterial`/standard glTF PBR APIs.
- Apply partial patches; unspecified values remain unchanged.
- Keep immutable original snapshots for reset.
- Implement copy-on-write for shared materials.
- Base-color/emissive textures are sRGB; normal/MR/occlusion are linear data.
- Preserve alpha mode and double-sided behavior unless a future explicit API changes them.
- Network/decode/upload failure must leave the current visible material intact.
- Coalesce duplicate texture requests; use ref-count/LRU semantics and memory budgets.
- Do not serialize GPU resources.

LIGHTING RULES

- Provide an out-of-box viewer-controlled studio lighting preset.
- Expose environment intensity, exposure, background, and a simple directional light configuration.
- Do not claim full GLB scene import when cameras/lights are not imported.

ASYNC, LIFECYCLE, AND SAFETY RULES

- Every load/session has a generation token.
- If load A completes after B starts, A must never replace B.
- Disposing the widget/controller during network/import/texture work must be safe.
- Do not blindly move `Node.fromGlbBytes()` or GPU creation into `Isolate.run`; GPU context/resources may be isolate-bound.
- Use streaming download progress, timeout, redirect limits, model byte limits, GLB header validation, and texture dimension/decoded-byte limits.
- Never log or serialize authorization headers.
- Web CORS failures must be typed and understandable.

RENDERING AND PERFORMANCE RULES

- Default to adaptive/on-demand rendering, not permanent ticking for a static scene.
- Maintain explicit render reasons such as initial load, gesture, inertia, material mutation, texture upload, resize, lifecycle resume, and one-shot request.
- If the upstream SceneView lacks a clean request-frame API, validate the smallest workaround and prefer a minimal upstream PR.
- Record download, import, indexing, texture, state-apply, and first-frame timings.
- Do not claim the package is faster than Filament, SceneKit, `interactive_3d`, BabylonJS, or WebView without comparable release/profile benchmarks.
- The product differentiators are common scene/material API, Flutter composition, web-facing API, assembly-aware addressing, and production lifecycle—not an assumed raw-rendering victory.

STATE RULES

- ViewerState is versioned.
- Persist model fingerprint, camera, visibility, material scalar descriptors, and texture source/cache descriptors.
- Never persist Node, Material, ui.Image, GPU texture, HTTP client, or secret headers.
- Applying state to a changed model returns a StateApplyReport with applied/missing/unsupported/failed entries.

TEST RULES

Use deterministic fixtures for:

- transform-only assembly nodes,
- duplicate names,
- multiple primitives,
- shared materials,
- PBR slots,
- missing UV,
- transparency/double-sided,
- orientation markers,
- unsupported topology/extension,
- malformed/oversized input.

Do not add skeletal or morph fixtures to MVP acceptance.

Build tests for stale loads, dispose during load, source replacement, repeated texture swaps, route reopen, cache eviction, idle frame stopping, and resource lifecycle.
