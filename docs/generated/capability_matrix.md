# Capability matrix

Generated placeholder. Update through tooling when adapter checks exist.

| Capability | Status | Notes |
| --- | --- | --- |
| Static GLB load | Planned | `Node.fromGlbBytes` adapter |
| Core PBR material read | Planned | `PhysicallyBasedMaterial` adapter |
| Runtime base color texture | Planned | Requires UV diagnostic |
| Metallic/roughness override | Planned | Patch + reset semantics |
| Alpha opaque/mask/blend | Supported | Runtime `MaterialAlphaMode` maps to `flutter_scene` alpha behavior for supported materials; unlit mask reports diagnostics because upstream treats it like blend |
| Material/effect masks | Blocked | Public opaque-family intent and validation exist, but rendering requires an opaque-family shader backend; current standard PBR path reports diagnostics rather than faking output |
| Authored material extension intent | Supported | Binary GLB reader maps transmission, IOR, volume, clearcoat, specular, and UV0 bufferView-backed extension texture bytes to internal patches; malformed values, duplicate node paths, missing UV0 texture slots, non-zero textureInfo texCoord, and unsupported KTX2/BasisU texture sources produce diagnostics with KTX2 header details when GLB bufferView bytes are available |
| Production shader preflight | Supported for custom shader backend | `productionShaders()` performs shader preflight and reports `backendKind: flutterSceneCustomShader` when required package shader entries are available; shader-load failures report typed diagnostics with `stage: shaderPreflight` |
| Transmission/glass | Production on verified iOS Simulator scope | Default policy is diagnostic-only; production policy routes supported glass patches through the repo-owned custom shader backend after preflight, with fixture, ToyCar, and acceptance metrics evidence |
| Clearcoat | Production on verified iOS Simulator scope | Diagnostic-only by default; production policy routes supported clearcoat patches through the repo-owned `.fmat` overlay backend after preflight, with synthetic matrix, ToyCar, and acceptance corpus evidence |
| Material extension platform evidence | Verified locally on iOS Simulator | iOS Simulator is the current verified production target for `backendKind: flutterSceneCustomShader`; macOS, Android, Web, and physical iOS are deferred/not run |
| V2 GLB capability preflight | Supported | Bounded GLB reader records required/used extensions, Draco primitive counts, meshopt compressed bufferViews, BasisU texture slots, material extension counts, imported texture roles, textureInfo UV sets, missing imported-UV diagnostics, and bounded KTX2 header summaries for GLB-embedded BasisU images before adapter import; meshopt bufferViews are rewritten before blocking decoder diagnostics |
| KHR_draco_mesh_compression | Dart rewrite wired; iOS native decode verified; Android native decode candidate-only | Required Draco assets fail before adapter import when no decoder is available; optional sibling `flutter_scene_viewer_draco` plugin vendors Google Draco 1.5.7 and links decoder-only source through Android CMake/iOS podspec, exposes opt-in diagnostics plus `decodeGlb`, has iOS and Android candidate bridges from compressed primitive manifests to `decodedPrimitives`, and includes Android JNI result marshaling; iOS Simulator evidence shows A1B32 rendering through native Draco decode, while Android NDK/SDK native app build evidence remains pending; root Dart sends compressed primitive manifests to native code, requires every decoded attribute and index payload, validates payload sizes, rewrites native `decodedPrimitives` into importer-ready GLB bytes, and re-runs preflight |
| EXT_meshopt_compression | Supported for embedded GLB bufferViews | Pure Dart pre-import decoder covers glTF `ATTRIBUTES`, `TRIANGLES`, and `INDICES` modes plus `NONE`, `OCTAHEDRAL`, `QUATERNION`, and `EXPONENTIAL` filters. The loader rewrites compressed bufferViews into standard embedded BIN bufferViews, removes `EXT_meshopt_compression` declarations, re-runs capability preflight, and only then calls the `flutter_scene` adapter. Unsupported or malformed meshopt metadata, external/non-BIN compressed buffers, and decode failures return typed `unsupportedModelFeature` rewrite diagnostics instead of silent fallback |
| KTX2 / KHR_texture_basisu | Supported through optional sibling transcoder plugin | Required BasisU/KTX2 texture slots produce typed `unsupportedModelFeature` diagnostics before adapter import with `basisuTranscodeUnavailable`, actionable `reason`/`nextStep`, and KTX2 header fields such as `vkFormat`, `levelCount`, and `supercompression` when the image bufferView is available. The root loader probes optional native `textureBasisu` capability, sends GLB KTX2 image bytes through a `basisuImages` MethodChannel manifest, accepts either rewritten GLB bytes or structured `decodedImages`, rewrites decoded PNG/JPEG bytes into ordinary GLB image bufferViews, removes `KHR_texture_basisu` declarations when no compressed texture references remain, and re-runs preflight before `flutter_scene` import. The optional `flutter_scene_viewer_basisu` plugin vendors Basis Universal plus Zstd, transcodes supported GLB-embedded KTX2 to RGBA32, encodes the result as PNG, and returns `decodedImages`. Missing plugin, disabled opt-in, native link failure, malformed payloads, oversized textures, and unsupported KTX2 layouts remain typed diagnostics instead of silent fallback. Local evidence includes host KTX2 transcode tests and an iOS Simulator smoke screenshot for a GLB-embedded KTX2 asset |
| Imported texture roles and UV sets | Supported for UV0 GLB bufferView core and extension textures | GLB texture slots are classified as color, normal, or data for core PBR, specular, transmission, and clearcoat slots; missing primitive UV sets are reported with source, mesh, primitive, material, UV set, and slot details; non-zero textureInfo texCoord produces diagnostics instead of UV0 fallback; bufferView-backed core PBR textures are returned as authored patches so the adapter reloads base-color/emissive as color, normal as normal, and metallic-roughness/occlusion as data; extension texture bytes are preserved for transmission, thickness, clearcoat, clearcoat roughness, clearcoat normal, specular, and specular color intent |
| KHR_materials_specular / KHR_materials_ior | Intent preserved, renderer diagnostics | Authored specular and IOR fields are parsed into serializable material intent and rejected with explicit capability diagnostics unless a backend advertises support |
| Part hierarchy | Planned | Node path + primitive index |
| Picking | Planned | Scene raycast adapter |
| Adaptive render | Planned | Widget/scheduler implementation |
| Skeletal animation | Deferred | Not MVP |
| Morph targets | Deferred | Not MVP |
| Imported lights/cameras | Deferred | Not MVP |
